Ansible常用模块

User模块

user 模块可以帮助我们管理远程主机上的用户，比如创建用户、修改用户、删除用户、为用户创建密钥对等操作。

常用参数

示例

1、在 ldap主机上创建名为 ding 的用户，如果用户已经存在，则不进行任何操作。

2、在 ldap主机上删除名为 ding 的用户，但是不会删除 ding 用户的家目录。

3、在 ldap 主机上删除名为 ding 的用户，同时会删除 ding 用户的家目录等信息。如果已经不存在 ding 的用户，则不做任何操作。

4、指定 ldap 主机上的 ding 用户的主组为 testgroup，testgroup 组需要提前存在，当不使用 group 设置主组时，默认主组与用户名相同。

5、指定ldap 主机上的 ding 用户的附加组为 testgroup2，testgroup2 组需要提前存在，当不使用 groups 设置附属组时，默认附加组与用户名相同。注意，为了保险起见，在不知道用户原来的附加组设定的情况下，最好将 append 参数设置为 yes，我们也可以一次性设置多个附加组，附加组之间用逗号隔开，比如groups=ding,testgroup,root 。

6、指定ldap 主机上的 ding 用户的过期时间为2019年12月31日。使用”date -d 2019-12-31 +%s“命令可以获取到对应日期的 unix 时间戳。

7.指定 ldap 主机上的 ding 用户的注释信息。

8、将 ldap 主机上的 ding 用户的密码设置为666666。

9、为 ldap上的 ding 用户生成 ssh 密钥对。默认在用户家目录的 ./ssh 目录中生成名为 id_rsa 的私钥和名为 id_rsa.pub 的公钥，如果已经存在同名密钥，并不会覆盖原来的密钥，即不做任何操作。

10、为 ldap 上的 ding 用户生成 ssh 密钥对，密钥对生成在 /opt 目录下，私钥名为 id_rsa_ding，公钥名为 id_rsa_ding.pub。

11、为ldap上的 ding 用户生成 ssh 密钥对，同时指定公钥中的注释信息为 “www.ding.com”，此参数只能在创建密钥时使用才会生效，并不能操作同名的老密钥。

12、为ldap上的 ding 用户生成 ssh 密钥对，同时指定私钥的密码为123456，此参数只能在创建密钥时使用才会生效，并不能操作同名的老密钥。

13、为ldap上的 ding 用户生成 ssh 密钥对，同时指定密钥对的类型为 dsa，当不显式指定密钥类型时，默认类型为 rsa，此参数只能在创建密钥时使用才会生效，并不能操作同名的老密钥。
